The Structure of the Input Method Table

When you open an Input Method Table which a text editor, it may look like this:

%gen_inp
%ename EHQ-Symbols
%cname 漁村符號
%encoding UTF-8
%selkey 1234567890
%keyname begin
....
%keyname end
%chardef begin
....
%chardef end

%gen_inp: Every table should start with this line. Please do not change it.

%ename: The English name of your Table-based Input Method Module. While it is blank, OpenVanilla will use the filename of the Input Method table insteed.

%cname: The Chinese name of your Table-based Input Method Module. It could be in Traditional Chinese or Simplified Chinese. While it is blank, OpenVanilla will use the English name insteed.

%selkey: The setting of the key to select the canidate characters from the candidate list. In most situations, users would like to use number keys from 1 to 0 to select a character. However, it could be customized.

You can also edit these settings by the "Basic Setting" pane of the OpenVanilla Generic Input Method Editor. 

The settings within "keyname" and "chardef" section is the core of an Input Method table. We will introduce them in next two sections: "The Key-mappings" and "The Character-Definitions" .
