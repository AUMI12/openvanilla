The Character-Definitions

The Character-Definitions defines the mapping between the keystroke combinitions and out characters. While using an Input method, user may type in several keys to compose the CJK character(s) that they want. The keystoke combinition means the sequence of the typed keys, and the final result is the output character.

While editing an Input Method Table of OpenVanilla, the keystroke combinitions should be defined with the keys but not the substitute characters. In other words, all characters in a keystroke combinition sould be a lower-cased letter or a half-width punctuation. A keystroke combinition sould not contain spaces too, because the space key is defined to terminate the process of composing one or more characters and start to display the Candidate Window, if there are two or more candidations of the output character(s).
