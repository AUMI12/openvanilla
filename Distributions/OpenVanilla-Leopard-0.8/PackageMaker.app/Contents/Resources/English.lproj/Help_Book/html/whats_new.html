What's New in PackageMaker

The following additions/enhancements have been made in PackageMaker 2.1:

Packages

  • Permissions Editor
  • Locator Support
  • File Filters
  • Requirements Editor
  • Resource forks now preserved by default
  • Authentication level now Administrator by default
    More Info…
  • Authentication level is changed dynamically to match the level needed for the default install location
  • The Install Fat binaries flag has been removed, as it was unused by Installer.app
  • Building a package fails if no bundle identifier is specified
    More Info…
  • Snapshot Packages
  • Package Verification

Metapackages

  • File Filters
  • Metapackage => Distribution conversion via “File > Convert Metapackage to Distribution” menu item
  • Metapackage Verification

Distributions

  • Localization Support
  • External Editor Support
  • Requirements Editor
  • Package authorization in the distribution is made to match that of the package for local packages
    More Info…
  • Choice scripts now use the same editing interface as global scripts
    More Info…
  • InstallationCheck and VolumeCheck fields are now combo-boxes, providing quick access to all defined global scripts
  • Distribution Verification
  • Added menu item “Project > Fill Package Reference Using Package…” which fills in the details of a package reference using a local package

General

  • View menu added with items to switch between interface panes
  • Package Setup Assistant
  • Improved CLI
    More Info…
  • Added this Help Book
  • Opening of .pkg and .mpkg files directly
